# 前端布局

## 核心原则

在编写 flex 或 grid 布局前，先判断布局意图：是单行、单列、二维网格，还是需要在不同宽度下重排。优先选择能直接表达布局关系的方案，避免用额外属性弥补结构不清。

## 选择 flex 或 grid

- 单行或单列布局优先使用 flex。
- 明确需要行列二维控制、卡片矩阵、复杂区域排布时使用 grid。
- 只需要排列多个相邻子元素时，不要为了简单间距引入 grid。

## 横向布局

- 使用横向布局时，明确标出哪些元素宽度可以弹性变化，哪些元素不应该被压缩。
- 使用 flex 时，为不需要弹性的元素设置 `flex-shrink: 0;`；使用 Tailwind 时使用 `shrink-0`。
- 一行 flex 中不要同时放入过多弹性元素，尤其是移动端或较窄页面。
- 只有在需要平分空间或按比例分配空间时，才让多个元素同时保持弹性。
- 文本、输入框、列表内容等可变区域通常是适合弹性伸缩的元素；图标、头像、固定宽按钮、标签、操作区通常不应被压缩。

## Flex 对齐与居中

- 保持 DOM 顺序与视觉顺序一致。不要直接使用 `flex-direction: row-reverse`、`justify-content: flex-end`（Tailwind 的 `flex-row-reverse`、`justify-end`）等反向或末端对齐属性调整普通布局。
- 需要将后续元素推至行末时，保持 `justify-content: flex-start`（默认可省略），并在两组元素之间添加一个明确的占位子元素，设置 `flex-grow: 1;`。
- 需要居中时，先判断能否通过 `padding-left`、`padding-right` 等内边距，让内容按正常起点对齐并满足视觉留白；不要为了居中固定父元素尺寸后再使用居中对齐。
- 通常仅当父元素尺寸和所需边距都无法预先确定时，才使用 `align-items: center`、`justify-content: center`（Tailwind 的 `items-center`、`justify-center`）。

## 垂直布局

- 垂直布局时，尽量不要默认拉伸子元素。
- 使用 flex column 时，优先让子元素从起点对齐，例如 `align-items: flex-start;` 或 Tailwind 的 `items-start`。
- 只有明确需要同宽、铺满或视觉对齐时，才让子元素横向拉伸。

## 间距

- flex 或 grid 的多个子元素之间优先使用 `gap`。
- 不要优先用子元素的 `margin` 制造同级元素间距。
- 只有在需要让单个元素产生特殊偏移、外部留白或兼容旧布局时，再考虑使用 `margin`。

## 显示顺序

- 正常情况下，通过调整 DOM 或组件中的元素顺序来改变显示顺序。
- 避免使用 `order`、`grid-area` 等属性只为修正错误的结构顺序。
- 只有在响应式布局确实需要同一组内容在不同断点呈现不同顺序时，才考虑使用顺序相关属性，并保持可读性和可维护性。

## 检查清单

- 是否已经说明每个横向子元素谁可以伸缩、谁不可以伸缩？
- 不应压缩的 flex 子元素是否设置了 `flex-shrink: 0;` 或 `shrink-0`？
- 单行或单列布局是否优先用了 flex？
- 多个同级子元素的间距是否优先用了 `gap`？
- 末端对齐是否保留了 `justify-start`，并通过 `flex-grow: 1;` 占位元素实现？
- 使用居中对齐前，是否已先评估通过内边距实现的可能性？
- 垂直布局是否避免了无意拉伸，并从起点对齐？
- 显示顺序是否优先通过结构调整完成？
